3.24 按摩师

题设

一个有名的按摩师会收到源源不断的预约请求，每个预约都可以选择接或不接。在每次预约服务之间要有休息时间，因此她不能接受相邻的预约。给定一个预约请求序列，替按摩师找到最优的预约集合（总预约时间最长），返回总的分钟数。

自己解

使用递归，构造两个数组，一个从nums[2]到最后的数，一个从nums[3]到最后的数，然后递归返回nums[0] + massage1(arr0) 和 nums[1] + massage1(arr1) 中的最大值。

java

最终只通过了80%，原因：时间和空间使用过大。

优解

抽象成斐波那契，递归的逻辑但是不用递归的方法，但不重新定义数组，只储存类斐波那契的最后两个数即可求出最后需要求的数。

java

时间和空间均超过100%

总结

  • 递归不一定非要使用方法的递归，只需要把递归的数据存储好同样可是完成递归操作，并且介绍时间空间的耗费
  • 使用递归方法时，尽量不要传入数组，否则资源消耗会很大。
  • 无必要的时间空间尽量减少，不是说非要做到完美利用资源，但是降一个维度其实比简优化已经是完美的算法要简单得多，正如这题，使用变量存储递归结果相比于传递数组进行递归就是一个降维，也仅仅是使用了变量存储而已，就从1%到达了100%，这样的降维，性价比太值了。
