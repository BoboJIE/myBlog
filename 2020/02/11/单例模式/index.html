单例模式
发表于 2020-02-11| 更新于 2020-02-17
阅读量:

单例模式：只实例化一个对象（保证一个类中只有一个实例）

举个例子：像window里面的回收站、任务管理器，打开两次，打开第二次时只是重新进入打开第一次时的窗口；还有手机里如果没有双开，只能登录一个微信或者QQ或者TIM，双开则破坏了单例模式

实现单例模式的方法有五种：

1. 饿汉式（线程安全，调用效率高，不能延时加载）

java

2. 懒汉式（线程安全，调用效率高，可以延时加载）

java

3. DCL饿汉式(Double Check，线程安全，调用效率高，可延时加载)

java

4. 静态内部类方式（线程安全，调用效率高，可延时加载）

java

5. 枚举方式（线程安全，调用效率高，不可延时加载）

java

解释 线程安全、调用效率、能否延时加载

1. 线程安全

CPU调度线程不可预测，有可能在get方法执行的时候被调度，从而中断方法，导致线程不安全

2. 调用效率

如果使用synchronized同步线程过多，会导致用时过长，效率低下

3. 延时加载

不在类加载时加载这个对象，而是在调用方法时加载这个对象，节省资源
