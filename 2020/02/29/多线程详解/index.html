线程、进程

平时我们看视频，是不是会有图像、声音、字幕，那么进程就是这个视频，线程就是视频中同时进行的图像、声音、字幕

线程与方法的区别

如图执行方法本质是一条线，而子线程则是开辟了另外一条线

线程的模拟

很多线程是模拟出来的，对于单核来说，在同一个时间，cpu只能执行一个代码，应为切换很快，会有多个线程同时执行的错觉。

线程的创建

  1. 创建Thread子类并继承Thread类
  2. 构造Thread子类对象
  3. 重写runnable接口方法
  4. 调用start()方法开启子线程（若使用run()方法则是调用方法，方法和线程是有上面述说区别的）
java

线程的调度

线程是由调度器的，调度器与操作系统紧密相关，不能被人为干预，具有不可预测性。

线程下载图片实现

利用common-io.jar包中的copyURLToFile(url, destination)方法下载网络图片。

  1. 下载common io包并粘贴进project目录，右键该包→add as library
  2. 创建类DownloadImage继承与Thread类
  3. 赋予DownloadImage类属性URL url;File destination;以便进copyURLToFile(url, destination)方法
  4. 构造方法中给url和destination赋值并调用start()方法开启线程
java

Runnable接口

使用Runnable接口也可启动多线程

  1. 使类实现Runnable接口
  2. 重写run()方法
  3. new Thread(p).start()开启线程，p为实现了Runnable接口的这个类

综上，使用多线程尽量使用Runnable接口

龟兔赛跑（并发）

  1. 创建赛道对象并实现Runnable接口
  2. 重写run方法
  3. new Thread(p，name)，p为赛道对象，name为参赛队员
  4. 让name为兔子的线程每百步sleep(10)
java

Callable接口

Callable接口必须重写call()方法，并且要给返回值。

java

静态代理

Thread类代理了{实现了Runnable接口的类}

条件：代理类与被代理类同时实现同一个接口

特征：代理类不仅要完成被代理类的操作，还要完成更多的操作

例如{实现了Runnable接口的类C}只需要完成Runnable接口的重写方法，然后丢给Thread类，Thread类不仅要完成Runnable方法，还要完成线程相关的操作

Lambda表达式

JDK8的新特性

演化过程：外部类→ 静态内部类→局部内部类→匿名内部类→Lambda表达式

java

线程的状态

  1. thread.state()查看线程状态
  2. thread.sleep(int duration)线程延时；thread.yield()线程礼让；thread.join()线程插入
  3. 修饰符synchronized给线程加锁
  4. synchroinized(Object obj) {代码块}，给代码块加锁，obj一般是变化的量。变化的量容易线程不安全。

CopyOnWrite容器

写时复制，即共享资源的时候先复制一个副本，最后替换

该容器很好地解决了读和写的关系，即读和写可以共存，在进行读操作时，应为写操作是在副本上进行的，不会影响读操作。

另外很好地解决了同步问题，若在方法上进行同步，则无论是否使用了容器，都会有同步，这样大大牺牲了效率，而CopyOnWrite容器则是只有在修改时才会进行同步，而读取不会同步，很好地解决了这个问题

Lock锁

抢票模型

java
  1. synchronized是隐式锁，Lock是显式锁，Lock有明显的开关锁操作
  2. Lock性能更优
  3. 性能Lock > synchronized代码块 > synchronized方法

生产者消费者问题

生产者→仓库→消费者

生产者→仓库线程生产产品进入仓库，并通知消费者生产了产品（notify()方法）

仓库→消费者线程取出产品，在仓库空的时候等待(wait()方法)

正常的线程是并行的

生产者消费者问题两线程互为条件

信号灯法

一个十字路口，线程代表x、y方向的车流，十字路口的交点两个线程不能同时使用，一个线程必须要等到另一个线程处于等待才能使用交点。利用flag标记可以有效解决这个问题，flag为true，x线程开启，等待两秒之后wait()，让出交点资源，y线程开启……

线程池

JDK 5.0线程池API：ExecutorService和Executors

ExecutorService：

子类ThreadPoolExecutor

方法void execute(Runnable command)，常用来执行Runnable

方法submit()，有返回值，常用来执行Callable

方法shutdown()，关闭线程池

Executors：

线程池的工具类，用于创建并返回不同类型的线程池
