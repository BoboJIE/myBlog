计算机网络概述

性能指标

传播时延（对应传播速度）：指电磁信号或者光信号在传输介质中传播一定的距离所花费的时间，与电磁或光的性质有关

传输时延（对应带宽）：指节点在发送数据时使数据块从结点进入到传输媒体所需的时间

就像长方形的水管，传播速度就是水的流速，带宽就是水管的很截面积，两者共同决定了水流量。

值得注意的是，传播速度是电磁波或光的物理特性和介质材质所决定，而这些因素变化不大，传播速度基本不会变。那么决定数据块输送快慢的就只有带宽了，大家经常看到下载东西时的速度也是这个带宽，单位M/s或kb/s，带宽中的单位Mb/s，M表示10^6，不是2^20，数据的大小Mb这里的M就代表2^20.

五层协议数据单元

应用程：报文

运输层：报文段

网络层：数据包（或分组）

数据链路层：数据帧

物理层：010101010101比特流

数据单元的封装

每一层都会在原来的封装基础上继续封装，俗称套娃，数据链路层较为特殊需要前后封装，既有头部又有尾部。

实体、协议、服务、SAP、接口

实体：发送或接收信息的硬件或软件进程

协议：两个对等实体通信行为规则的集合

服务：同一开放系统中某层向其相邻上一层提供的操作并屏蔽细节具体实现

SAP：同一系统中相邻两层实体的交互处

粉红色箭头代表服务，蓝色方块代表一个个的SAP，橙色方体代表接口，接口除了SAP还有其他的。

面向连接与无连接服务

面向连接服务：双方都可进行通信，即两者都在线，特性可靠、可控，如TCP协议

面向无连接服务：双方仅一方在线，一方作”尽最大努力交付“的准备，两个实体之间不需要事先建立好一个连接，特性不可靠，不可控。

因特网标准化工作

所有的因特网标准以RFC（Request For Comment）形式在因特网上发布。

RFC文档有实验RFC、提供信息的RFC。

RFC文档建立过程：实验RFC形成建议标准→草案标准→因特网标准→正式使用的RFC

物理层

四个技术特性

  1. 机械特性：规定了物理连接时接口所需连接插件的形状、规格尺寸、引脚数量、排列情况以及固定和锁定装置等
  2. 电气特性：规定了DTE和DCE间导线的电气连接及有关电路方面的特性，例如0的电压特性和1的电压特性
  3. 功能特性：定义了DTE和DCE间各信号线的功能，即某条信号线上出现某一电平的电压表示何种确切含义
  4. 规程特性：定义了利用信号线进行二进制比特流传输的一组操作过程，即在建立、维持、拆除物理连接及信息交换过程中，各信号线的工作规则和先后顺序

DTE和DCE

DTE（Data Terminal Equipment）：数据终端设备

计算机和路由器属于DTE设备

DCE（DATA Circuit-terminating Equipment）：数据电路端设备，DCE提供DTE和传输线路之间信号变换和编码的功能。

modem、交换机、集线器属于DCE设备

RJ-45（水晶头）

  1. 8芯模块化插孔或插座
  2. 广泛用于局域网和ADSL宽带上网用户
  3. 两种排序方式：T568A、T568B

)

  1. 同为DTE或DCE设备使用交叉方式连接，DTE和DCE之间使用直接连接方式

异步传输与同步传输

  1. 异步传输通过捉取起始码和终止码实现再次同步，面向字符，单位字符，头部信息之前始终位于同一V电平，当检测到跳变则识别为起始码，当识别了一个字符的比特信息后，则识别终止码V电平终止传输
  2. 同步传输则是抽取同步信号，面向比特，单位帧，头部信息之前始终电平始终以同一频率跳变，使接收端与数据时钟同步，便于识别01信号。
  3. 同步传输较于异步传输效率高
